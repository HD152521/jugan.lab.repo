# w04 - 이번 달 배달비 계산기

> 이 폴더는 **이번 주 목표·계획**. 실제 앱 코드는 `apps/baedalbi`(예정)에 둔다.
> 타깃: 자취생 · hub 실험 #4

## 한 줄 컨셉
카드 결제 내역을 붙여넣으면 **배달비만** 골라 합산하고, "치킨 N마리"로 환산해 현실을 보여준다.

## 관심 끄는 축 (둘 다 노림, 주력 = 바이럴)
- 🚀 **바이럴(주력): 충격 환산 + 자조 개그** — "이번 달 배달비 8만원 = 치킨 4마리 🍗" 카드. 공감·현타로 공유됨.
- 🔁 **리텐션(보조): 월별 추적** — 매달 붙여넣어 이번 달 vs 지난 달, 목표("5만원 이하") 관리.

## 이번 주 목표
- "내 배달비가 이 정도였어?"를 붙여넣기 한 번으로 체감하게.
- 붙여넣기 파싱 → 배달비 합산 → 환산 카드까지 한 흐름 완성.

## 핵심 기능
### 붙여넣기 파서 (핵심 난관)
- 카드사 문자/앱 결제 내역 텍스트를 붙여넣으면 **배달앱 결제만 추출**
- 인식 키워드: 배달의민족/배민, 쿠팡이츠, 요기요, 땡겨요 등 (+ 사용자가 키워드 추가)
- 금액·날짜 파싱 → 합산. 인식 실패 줄은 "확인 필요"로 따로 표시(수동 포함/제외 토글)
- **전부 브라우저 안에서만 처리, 서버 전송 없음** 명시 (민감정보 신뢰)

### 환산 재미 (바이럴 훅)
- 합계를 실물로 환산: 🍗 치킨 N마리 / ☕ 스벅 N잔 / 🎬 영화 N번 / 📺 넷플릭스 N개월
- "이번 달 배달비 리포트" 공유 카드(canvas PNG, 연차술사 패턴 재사용)

### 월별 추적 (리텐션 훅)
- 월별 합계 저장 → 이번 달 vs 지난 달 증감(%), 간단 막대 추이
- 목표 금액 설정 → 초과 시 경고("치킨값으로 벌써...")

## 만드는 순서
1. 파서(`parser.js`) 순수 함수 — 텍스트 → [{앱, 금액, 날짜}] , 테스트 가능하게
2. 합산·환산 로직 + 단가 데이터(`data.js`)
3. UI: 붙여넣기 → 결과(포함/제외 토글) → 환산 카드
4. 월별 저장(localStorage) + 추이/목표
5. 공유 카드

## 데이터/기술 메모 & 리스크
- **파싱이 최대 리스크**: 카드사·앱마다 텍스트 포맷 제각각 → 완벽 대신 "대표 포맷 + 수동 보정(포함/제외 토글)"로 커버
- 단가(치킨 2만원 등)는 상수로, 물가 바뀌면 조정. 재미 위주라 정밀할 필요 없음
- 배달비만 vs 음식값 포함? → v1은 "배달앱 결제 총액" 기준으로 단순화, 문구로 명확히
