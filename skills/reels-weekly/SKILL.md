# 주간 릴스 파이프라인 (reels-weekly)

`agents/reels-*` 5종을 **메인 세션이 순서대로 오케스트레이션**하는 러너다.
서브에이전트는 다른 서브에이전트를 호출할 수 없으므로, 단계 간 연결·게이트 통과 판단은 이 스킬을 실행하는 메인 세션이 책임진다.

## 전제 (실행 전 확인)

1. **에이전트 인식**: `agents/reels-*.md`가 `.claude/agents/`에 복사돼 있어야 Task로 호출 가능.
   - 없으면 먼저: `Copy-Item agents\reels-*.md .claude\agents\` 안내 (프로젝트 레벨).
2. **대상 주차**: 인자로 주차를 받는다 (예: `/reels-weekly w05`). 없으면 `weeks/` 최신 폴더를 후보로 제시하고 확인.
3. **산출물 폴더**: `weeks/<주차>/promo/` — 없으면 생성. 모든 산출물이 여기 쌓인다.
4. **촬영 대상**: 이번 주 실험 앱(`apps/<slug>/` 또는 `weeks/<주차>/code/`) 경로를 파악해 producer에게 넘길 준비.

## 파이프라인 (4 스테이지 + 스테이지마다 director 게이트)

각 스테이지는 **담당 에이전트 실행 → director 검수 게이트** 순서. 게이트 판정이 "승인/조건부 승인"이어야 다음 스테이지로 넘어간다.

### 스테이지 1 — 레퍼런스 분석
1. **사용자에게 레퍼런스 릴스 3~5개를 먼저 받는다**(노션 파이프라인 1번). 없으면 요청하고 대기 — 블로그로 대체 금지.
2. `reels-trend-scout` 실행. 프롬프트에 대상 주차·실험명 + 레퍼런스 릴스 전달. 산출물: `weeks/<주차>/promo/레퍼런스리포트.md`.
3. **게이트 1**: `reels-director` 실행, 레퍼런스리포트.md 검수. 핵심 질문 — 실제 릴스 기반인가(블로그·마케팅 글이면 반려)? 훅 유형이 태깅됐나?

### 스테이지 2 — 기획
1. `reels-strategist` 실행. 입력: 트렌드리포트.md + 이번 주 실험 README. 산출물: `기획서.md`.
2. **게이트 2**: director 검수 — 첫 1.5초 훅이 구체적인가? 트렌드 근거 인용됐나? 저장/공유 장치 있나?

### 스테이지 3 — 대본
1. `reels-scriptwriter` 실행. 입력: 기획서.md. 산출물: `대본.md`.
2. **게이트 3**: director 검수 — 소리 없이 자막만 봐도 이해되나? 첫 씬 결론부터인가? 씬 시간 합계 검산 맞나?

### 스테이지 4 — 영상 제작
1. `reels-producer` 실행. 입력: 대본.md + 촬영 대상 앱 경로. 산출물: `<실험명>_릴스.mp4` + `제작로그.md`.
   - 파이프라인 스크립트는 `shared/video-pipeline/`. 없으면 producer가 새로 작성(재사용 가능하게).
2. **게이트 4 (최종)**: director 검수 + **업로드 체크리스트**. 훅이 첫 프레임에 실제 보이나? 해상도/오디오/길이 검증 통과했나?

## 게이트 처리 규칙

- 판정은 매번 `weeks/<주차>/promo/검수.md`에 누적 기록(director가 씀).
- **반려 시**: director의 "수정 지시"를 그대로 해당 에이전트에게 다시 넘겨 **최대 1회 왕복**으로 재실행. 2회째도 반려면 사용자에게 판단을 넘긴다(어중간한 콘텐츠가 제일 비싸다).
- **조건부 승인**: 경미한 수정은 다음 스테이지 진행하되 검수.md의 지적사항을 producer/최종 단계에서 반영.

## 병렬화 주의

이 파이프라인은 **엄격한 순차**다(각 산출물이 다음 입력). 스테이지를 병렬로 돌리지 말 것.
단, 트렌드 조사가 오래 걸리면 그동안 촬영 대상 앱 경로·promo 폴더 준비 같은 부수 작업은 병행 가능.

## 완료 보고

파이프라인 종료 시 메인 세션이 요약한다:
- 생성된 산출물 목록(경로)과 각 게이트 판정
- 최종 업로드 체크리스트 상태(미완 항목 명시 — 예: 인스타 앱에서 트렌딩 오디오 얹기, hub `experiments.js` status/url 반영)
- mp4는 `.gitignore` 대상이라 커밋 안 됨을 상기

## 브랜드 불변 규칙 (전 스테이지 공통, director가 강제) — 출처: JuganLab 노션 릴스 킷

- **마스코트 = 안경 오리 (필수 주연)**: 노란 몸통 + 동그란 안경, 크루드 손드로잉. 표정으로 감정 서사. 오리 없으면 반려.
- **팔레트: 딥그린 메인 + 옐로우 포인트** (민트 아님). 표지 "실험 #NN" 프레임.
- **목표 = 알고리즘 도달**(앱 소개 아님). 화면넘기기 금지, 공감·상황극·페인 서사로 후킹. "이게 불편해서 만듦" 톤.
- **톤: 개발자 반말·밈·살짝 자학**, 짧게. 훅 급소 = 소리 끄고 첫 2초에 안 넘기게.
- 캡션 첫 줄 = 페인/결과부터, 마지막 = CTA(댓글 유도). BGM 미포함(인스타 앱 트렌딩 오디오로 도달 혜택).
