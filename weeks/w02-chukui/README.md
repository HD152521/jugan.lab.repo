# w02 - 축의금 계산기

> 이 폴더는 **이번 주 목표·계획**. 실제 앱 코드는 `apps/chukui`(예정)에 둔다.
> 타깃: 2030 · hub 실험 #2

## 한 줄 컨셉
"이 결혼식/경조사, 나는 얼마 내야 하지?"를 1초에 답해주고, 낸 돈·받은 돈을 자동으로 장부에 쌓아준다.

## 관심 끄는 축 (둘 다 노림, 주력 = 리텐션)
- 🔁 **리텐션(주력): 경조사 장부** — 사람들은 실제로 축의금을 수첩/엑셀로 관리한다. 이걸 앱으로 옮기면 경조사 있을 때마다 다시 온다.
- 🚀 **바이럴(보조): "얼마 내지?" 추천 결과 공유** — "직장 동료 + 참석 + 호텔 → 10만원" 같은 애매한 상황의 답을 카드로 공유.

> **핵심 결정**: 장부는 평생 쌓이는 데이터라 다른 실험과 달리 **지속성이 본질**. 로그인 없이 "계정 없는 클라우드(복구코드/익명키)"로 폰 바꿔도 유지되게 한다. (다른 3개 실험은 로컬 저장으로 충분 — 순간 소비형)

## 이번 주 목표
- 축의금 "적정 금액"의 애매함을 없애고, 경조사 수지를 한눈에 보게 한다.
- 최소 기능으로 "추천 → 장부 기록 → 다음에 받은 만큼 돌려주기"까지 한 흐름 완성.

## 핵심 기능
### 추천기 (바이럴 훅)
- 입력: **관계**(가족/친한친구/친구/직장동료/지인/거래처), **참석 여부**(참석·식사O / 참석·식사X / 불참), **식장 등급**(예식장/호텔), **친밀도 슬라이더**
- 출력: 추천 금액 + **근거 한 줄** + "애매하면 X~Y원" 범위
- 규칙 기반: 기본 시세(3/5/10/20만) × 참석·식대·관계 가중치. 홀수 관례(3·5·7·10) 반영
- 공유 카드: "이 경조사, N만원 추천" 이미지/링크 (연차술사 이미지 카드 패턴 재사용)

### 경조사 장부 (리텐션 훅)
- **낸 돈 / 받은 돈** 기록 (이름·관계·금액·경조사 종류·날짜)
- 사람별 **수지**: "김철수 → 내가 10만원 냄 / 받은 적 없음", "이영희 → 받은 5만원, 아직 안 냄 ⚠️"
- 추천기와 연동: 그 사람에게 **받은 기록이 있으면** "예전에 5만원 받았으니 최소 5만원" 자동 반영 ← 핵심 연계
- 총계: 올해 낸 축의금 합계 / 받은 합계 / 순수지

### 데이터 지속성 — 계정 없는 클라우드 (내구성 B) ★ 이 앱의 필수 요소
- 장부는 **평생 쌓이는** 데이터 → localStorage만으론 폰 교체·캐시삭제·iOS ITP(7일)로 유실. 여기가 유일하게 내구성 투자가 필요한 실험.
- **로그인 UI 없음**: 첫 사용 시 익명 키 자동 발급 → 클라우드 저장. 사용자는 **복구코드**(또는 패스키)만 저장해두면 어느 기기서든 복원.
- 서버엔 **이름 없이 코드로만** 저장 → "가입 마찰 0인데 폰 바꿔도 유지" + 신뢰 유지.
- 기술: Supabase 익명 auth(또는 복구코드 → serverless KV). 다른 3개 실험과 달리 **가벼운 백엔드 1개** 추가.
- 로컬 우선(local-first): 오프라인에서도 즉시 동작, 온라인 되면 동기화. 신뢰 위해 "무엇을 서버에 두는지" 명시.

### 리마인더 (리텐션 강화)
- "곧 갚아야 할 사람"(받았는데 아직 안 낸) 목록 상단 고정
- 지인 결혼 예정일 등록 → 캘린더(ICS) 등록 / D-day

## 만드는 순서
1. 시세 데이터·가중치 규칙 확정 (`data.js`) — 2026 관계별 평균 리서치
2. 추천 순수 함수 (`recommend.js`) — 입력 → 금액, 테스트 가능하게
3. 장부 상태/저장 — **local-first**: 우선 localStorage로 동작 완성(추가/수정/삭제·수지)
4. UI: 추천 탭 / 장부 탭 (연차술사 모드 전환 패턴 재사용)
5. **계정 없는 클라우드 동기화** — 익명키 발급 + 복구코드, Supabase 익명 auth(또는 serverless KV). 로컬 데이터를 클라우드에 미러링
6. 공유 카드 + 캘린더 리마인더

- [ ] 로컬 우선 동작(오프라인 OK) + 온라인 시 동기화

## 데이터/기술 메모 & 리스크
- 시세는 지역·물가·관계 해석이 갈려 정답이 없음 → "추천 + 범위 + 근거"로 제시(단정 X)
- **지속성 = 이 앱의 사용 이유.** 계정 없는 클라우드로 폰 교체·캐시삭제에도 유지. 서버엔 실명 대신 사용자가 붙인 라벨/코드만 → 유출 리스크 최소화
- 복구코드 분실 시 복원 불가 → "코드 꼭 저장/캡처" 안내를 발급 시점에 강조
- 백엔드 최소화: RLS/익명 auth 기본 설정 준수, 개인정보(이름·금액) 취급이므로 보안 리뷰 1회 필수
