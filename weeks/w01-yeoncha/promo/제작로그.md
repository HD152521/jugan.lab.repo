## 제작 파이프라인 실행 순서 (재현 커맨드)
```bash
cd shared/video-pipeline
npm install                       # playwright, ffmpeg-static, ffprobe-static, msedge-tts
npx playwright install chromium

node server.js &                  # http://localhost:4173 (apps/yeoncha 서빙)
node tts-gen.js                   # out/narration/line1~6.mp3 + narration.json
node capture.js                   # out/frames/scene*.png, cover.png
node build.js                     # out/segments/*, out/final/*, weeks/w01-yeoncha/promo/*로 복사
```

## 단계별 성공/실패 기록
- **npm install (playwright / ffmpeg-static / ffprobe-static / msedge-tts)**: 성공. `msedge-tts` 최신 버전(2.0.7)이 정상 동작해 대안 패키지(edge-tts 계열) 전환 불필요했음.
- **playwright install chromium**: 성공 (`chromium-1228`, 캐시 확인 완료).
- **정적 서버**: node 내장 http 모듈로 10줄 서버 작성, background 프로세스로 유지. localStorage 의존 때문에 file://는 처음부터 배제하고 http://localhost:4173 사용.
- **TTS 생성**: ko-KR-SunHiNeural, mp3 96kbps로 6줄 모두 1회 성공. ffprobe로 각 파일 길이 측정 → narration.json에 기록. (사소 수정: 초안 스크립트에 "바뀌"로 오타가 있던 것을 발견해 "바뀜"으로 즉시 정정 후 생성 — 산출 음성에는 오타 없음.)
- **화면 캡처**: 진짜 조작 기반 방식 채택(실 video 녹화가 아닌 스틸 타임라인 방식으로 처음부터 설계 — Playwright `page.screenshot()`을 viewport 540x960 · deviceScaleFactor 2로 찍어 1080x1920을 별도 스케일 없이 바로 확보). 상태는 `localStorage`(yeonchasulsa:v1)에 실제 앱 스키마 그대로 주입 + 씬3는 실제 클릭/타이핑으로 조작. 자막·배지는 DOM에 오버레이 엘리먼트를 추가해 스크린샷에 함께 구움(ffmpeg drawtext 미사용, 지시대로 브라우저 렌더링에 의존).
  - 1차 캡처 후 검수에서 씬4 자막이 "내 캘린더"까지 민트로 잘못 강조되고, 씬3c 자막이 스펙에 없는 "토글 누르면…" 문구로 바뀐 것을 발견 → `capture.js` 수정 후 재캡처, 스펙과 100% 일치 확인.
- **연휴 후보 재검증**: `node -e`로 optimizer.js/holidays.js를 직접 호출해 leave=1·leave=3 각각 top6을 뽑아, year='2026' 필터를 걸었을 때 BEST 카드가 게이트3 확정값(5일/D-75, 9일/D-71)과 정확히 일치함을 확인 후 캡처 상태(state.year='2026')를 그렇게 고정.
- **합성(ffmpeg-static)**: 스틸마다 `-loop 1 -t <sec>`로 개별 h264 세그먼트를 만든 뒤 concat demuxer로 이어붙임(스케일 로직 포함하지만 원본이 이미 1080x1920이라 실질적인 리스케일은 발생하지 않음). 나레이션은 6개 mp3를 씬 시작 시각으로 `adelay`한 뒤 `amix`로 한 트랙에 합쳐 비디오와 mux. BGM은 지시대로 넣지 않음.
- **검증(ffprobe)**: 해상도 1080x1920 확인, 오디오 트랙(aac) 존재 확인, 컨테이너 길이 19.84s(목표 18~22s 이내) 확인, 첫 프레임 추출 결과 히어로 카드(5일 연휴/D-75/SS 갓성비)가 프레임 0부터 그대로 보이는 것 확인.
- **실패한 것 없음.** 모든 단계가 최초 계획한 스틸 타임라인 방식으로 한 번에 완주됨 — 실제 video 녹화 폴백이 필요하지 않았음.
